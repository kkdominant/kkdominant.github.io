1 hexo的Matery主题代码块显示问题

你的代码块是否显示为这样?

尝试过各种方法都没有效果、比如卸载插件,和下面的方法都没有效果代码链接

highlight:                      # 代码块的设置
  enable: false                 # 开启代码块高亮
  line_number: true             # 显示行数
  auto_detect: false            # 如果未指定语言，则启用自动检测
  tab_replace: ''               # 用 n 个空格替换 tabs；如果值为空，则不会替换 tabs
  wrap: true
  hljs: false
prismjs:
  enable: true
  preprocess: true
  line_number: true
  tab_replace: ''

有没有发现highlight:上面还有一行？

syntax_highlighter: highlight.js

我们需要修改这一行、将highlight.js改为prismjs才行

只需要做如下修改即可、因为我们的primjs应该都是v7.0.0及以上：

# _config.yml
syntax_highlighter: prismjs
prismjs:
  preprocess: true
  line_number: true
  line_threshold: 0
  tab_replace: ''

官网对这个问题的说明

然后就显示正常了

设置valine评论时显示qq头像

找了好久也没有搜到、最后我在这篇文章里找到了2 valine 评论系统使用教程我看matery主题里也没有enableQQ这一项
解决方法:
在hexo-theme-matery —— layout—-partial—-valine.ejs文件里(或者直接Ctrl+P搜索valine.ejs)在大概251行的位置加上

enableQQ: true,

就可以了、注意后面的逗号

将网站提交到谷歌搜索(hexo-theme-matery主题)

进入网站/search-console
填写的自己的网站地址3:

选择推荐验证方式、下载html文件到:
hexo-theme-matery—source里、如果你的主题是next就是上传到hexo-theme-next—source里

在git执行:

hexo cl & hexo g -d

等待一分钟:

这样就验证成功了

提交到必应

也是和谷歌一样的操作方式、下载文件到hexo-theme-matery—source里即可然后

hexo cl & hexo g -d

等一会就可以去验证了、虽然这里是xml文件但验证方式和html文件没有区别
记得提交网站地图

然后请求编制索引:

站点地图申请

npm install hexo-generator-sitemap --save
hexo hexo cl & hexo g -d

此时在public目录下可见sitemap.xml，即已完成站点地图的生成4。
然后去/search-console提交即可、填入 sitemap.xml点击提交即可
